Objective
Research and determine the best design for automatically restarting the EPA (IBM Planning Analytics Cloud) instance on weekends — to proactively reclaim memory, reduce performance degradation, and improve platform stability.
IBM Official Response — Apr 23, 2026
IBM Support Engineer · Jakub Makowka · Case Response April 23, 2026
✕Blocked
Scheduled / Automatic Restart by Customer

Not supported in current PA Cloud versions. The Windows Task Scheduler approach that was available in older versions has been removed and is no longer supported by IBM.

✕Blocked
Additional IBM-Managed Maintenance Windows

Not possible beyond IBM's standard maintenance cadence. Customers cannot request extra maintenance windows to trigger additional restarts.

✕Blocked
Programmatic Restart via REST API / PAW

Only Start/Stop operations are available via the PAW Administration API. The REST endpoints do not have the required permissions to restart the underlying Windows service. A full restart cannot be triggered programmatically.

✓Confirmed
IBM Weekly Maintenance Includes Data Tier Restart

IBM confirmed that during their weekly maintenance windows, Data Tiers are usually restarted. This can be verified by reviewing Server Logs in PAW after each maintenance window.

Key Positive Finding
⭐ Unexpected Outcome

While all customer-triggered automated restart paths are confirmed blocked, IBM's existing weekly maintenance cadence already includes a Data Tier restart — which is precisely the memory reclaim mechanism we were looking for. This means EPA is very likely already benefiting from a weekly memory reset through IBM's standard operations. We can verify this immediately by reviewing PAW Server Logs after IBM's next maintenance window.

Final Conclusion
SPIKE #3981779 — Conclusion

All customer-controlled automated restart options are architecturally blocked in IBM PA Cloud SaaS — this is a platform-level constraint, not a configuration gap. However, IBM's own weekly maintenance already delivers the core memory reclaim benefit we set out to achieve. The recommended path is to verify this via Server Logs, monitor RAM trends, and supplement with manual PAW restarts only if IBM's maintenance cadence proves insufficient for our workload.
